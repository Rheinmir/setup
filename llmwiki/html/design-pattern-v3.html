01 · Microservices

Monolith vs Microservices

Microservices không phải silver bullet — chúng giải quyết vấn đề team scalability và deploy independence, nhưng thêm complexity đáng kể.

Communication Patterns

  • Sync REST/gRPC: cần response ngay
  • Async messaging: fire-and-forget, decouple
  • Event sourcing: audit trail, replay events
  • Saga: distributed transaction — choreography hoặc orchestration

Monolith trước, Microservices sau

  • Giữ monolith: team nhỏ, domain chưa ổn
  • Tách khi: team lớn, deploy freq khác nhau
  • Strangler Fig: tách từng module dần
  • Domain-Driven Design giúp xác định service boundary
02 · Caching

Caching Layers & Strategies

Cache đúng chỗ, đúng chiến lược giảm latency đáng kể và giảm tải DB. Cache sai → stale data, inconsistency.

Cache-Aside (phổ biến nhất)

  • Read: check cache → miss → read DB → write cache → return
  • Write: invalidate cache → write DB
  • Chỉ cache data được đọc thực sự
  • Risk: cache stampede khi nhiều miss đồng thời

Write-Through vs Write-Behind

  • Write-Through: write cache + DB sync — consistency tốt, write chậm hơn
  • Write-Behind: write cache → async flush DB — write nhanh, risk mất data
  • Cache invalidation vẫn là hard problem
  • TTL + version key để handle stale data
03 · High Availability

Circuit Breaker & Bulkhead

Hệ thống resilient không tránh failure — nó fail gracefully. Circuit Breaker và Bulkhead là hai pattern cốt lõi để làm điều đó.

Bulkhead Pattern

  • Tách resource pool cho từng downstream service
  • Service B chậm → không drag cả Service A
  • Thread pool isolation per dependency
  • Impl: Resilience4j, Polly, Hystrix (deprecated)

Retry với Jitter

  • Exponential backoff: 1s → 2s → 4s
  • + Random jitter (0–1s) tránh thundering herd
  • Max retry: 3–5 lần, sau đó fail fast
  • Chỉ retry idempotent operations (GET, PUT)
// CQRS — tách Read/Write model
Write side: Command → Write Model → Event Store (Kafka)
Read  side: Query  → Read Model (optimized projection)

// Stateless service — scale tự do
Session → Redis (shared state)
Không lưu state trong process memory
Mọi instance đều serve được request

// Event-driven
Producer → Event Bus → Consumer(s)
Fan-out: nhiều consumer xử lý cùng 1 event
→ Tách coupling, dễ thêm feature mới
