03 · bài học 230626

2 bẫy orca CLI — gốc rễ của orca_guard

Phiên này vấp thật 2 lỗi khi lái orca orchestration. Cả hai đều "im lặng" — orca trả ok:false hoặc resolve gate trỏ task ma, không la lỗi rõ. Chính chúng đẻ ra quyết định "chặn/inject bằng máy".

Bẫy 1 — status enum

  • task-update --status done → ok:false lặng lẽ
  • Status hợp lệ: ready · in_progress · completed · failed
  • Không có done — phải là completed
  • Giờ orca_guard chặn thẳng tại chỗ

Bẫy 2 — hai id

  • task-create --json trả envelope id (uuid) và result.task.id
  • Gate/dispatch/update phải dùng result.task.id
  • Dùng nhầm envelope → gate trỏ task ma, task kẹt ready
  • Giờ orca_guard inject nhắc khi thấy task-create
# SAI (đã vấp):
TID=$(echo "$RESP" | jq .id)            # envelope id ✗
orca ... task-update --status done       # enum sai ✗ → ok:false
# ĐÚNG (bài học):
TID=$(echo "$RESP" | jq .result.task.id) # ✓
orca ... task-update --status completed   # ✓
