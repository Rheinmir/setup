01 · Mô hình

1 lõi, nhiều dây cắm — và 2 tầng do policy quyết định

Luật ở policy.yaml → lõi đọc & áp → adapter từng vendor gọi vào (hoặc đọc config sinh từ policy) → CI là sàn. Tầng session vs repo không hard-code, mà do enforce_at của từng luật.

Ví dụ tầng: no_write_raw = [session] → agent bị chặn ghi raw/, nhưng con người vẫn commit raw/ hợp lệ (repo không áp). require_origin = [session, repo] → áp cả hai.

02 · Nguồn chân lý

policy.yaml — luật ở một chỗ, dày tùy ý

Mỗi luật có kind (quyết định handler nào trong lõi áp) + enforce_at (tầng nào). Thêm luật cùng kind = chỉ sửa file này, KHÔNG đụng code adapter.

version: 1
rules:
  no_write_raw:
    id: R1
    kind: deny_write          # LOẠI → handler chặn ghi path
    statement: "raw/ là inbox của con người — agent chỉ đọc."
    enforce_at: [session]      # KHÔNG repo → người commit raw/ vẫn hợp lệ
    deny_write_globs: ["**/raw/**", "raw/**"]

  require_origin:
    id: R2
    kind: require_section     # LOẠI → file khớp glob phải chứa heading
    enforce_at: [session, repo]
    require_section: "## Origin"
    target_globs: ["**/wiki/concepts/**/*.md", "**/wiki/entities/**/*.md",
                   "**/wiki/sources/**/*.md", "**/wiki/draft/**/*.md"]
    exclude_basenames: ["README.md", "index.md", "log.md"]

2 kind lõi hỗ trợ

  • deny_write — chặn ghi path khớp deny_write_globs (+ lệnh bash ghi vào đó)
  • require_section — file khớp target_globs (trừ exclude_basenames) phải chứa require_section

Khớp linh hoạt / chặt biên

  • ## Origin ✓, ## Origin (2 space) ✓
  • ## Origins ✗, ### Origin (h3) ✗ — chặt ở biên
  • glob khớp theo segment: coleslaw.md/raws/ KHÔNG dính raw/
03 · Lõi

llmwiki-validate.py — 3 mode, exit code, fail-open

Lõi vendor-neutral. Mỗi caller chọn mode phù hợp; exit code khác nhau theo tầng để session (chặn) và repo (fail commit) phân biệt được.

LệnhTầngInputExit
path <FILE>session1 path ghi2 = chặn · 0 = ok
claude-hooksessionPreToolUse JSON (stdin)2 = chặn · 0 = ok
files <F...>reponhiều path trên disk1 = vi phạm · 0 = ok
 
# session — kiểm 1 path (opencode plugin / quick check)
python3 bin/llmwiki-validate.py path llmwiki/raw/x.md   # → exit 2 (chặn)

# session — converter Claude: đọc PreToolUse JSON từ stdin
echo '{"tool_name":"Write","tool_input":{"file_path":"llmwiki/raw/x.md"}}' \
  | python3 bin/llmwiki-validate.py claude-hook          # → exit 2

# repo — pre-commit / CI quét file đổi
python3 bin/llmwiki-validate.py files wiki/concepts/a.md wiki/entities/b.md  # → exit 1 nếu thiếu Origin

fail-open (exit 0) khi

  • thiếu pyyaml / không đọc được policy
  • JSON stdin hỏng, mode lạ
  • Lý do: lỗi hạ tầng KHÔNG được chặn người dùng — sàn repo (CI) vẫn đỡ

claude-hook = converter Claude

  • Write/Edit/MultiEdit → kiểm file_path + content
  • Bash → kiểm command (redirect/tee/touch/cp→raw)
  • Vendor khác có converter tương tự, cùng gọi lõi

Nội bộ lõi (chi tiết)

Quy ước exit code

  • exit 2 cho session: Claude PreToolUse coi 2 = "block tool", stderr hiện cho agent đọc
  • exit 1 cho repo: pre-commit/CI coi non-zero = fail (tách khỏi 2 để phân biệt nguồn)
  • exit 0 / fail-open: thiếu pyyaml, không đọc được policy, JSON hỏng, mode lạ → lỗi hạ tầng KHÔNG chặn người dùng; sàn repo vẫn đỡ

Glob engine & bash surface

  • glob_to_regex: ** xuyên thư mục · * trong 1 segment · ?
  • Khớp theo segment → coleslaw.md/raws//myraw/ không dính **/raw/**
  • Bash deny soi bề mặt: bắt >/>>, tee, touch, sed -i<sfx> <file>, cp/mv/rsync đích raw/ — KHÔNG bắt python -c, rm (xem GAPS §06)
04 · CI & nhánh

CI là sàn · version cũ backup, bản mới lên orca

Đã thực hiện: backup state cũ ra nhánh riêng, commit bản mới lên orca, push, CI chạy xanh cả 2 job.

# 2 job trong .github/workflows/harness.yml (on: pull_request + push orca)
selftest:          pip install pyyaml; bash demo.sh; bash test-broad.sh   # 67 assertion mỗi PR
validate-content:  git diff base..HEAD → file .md đổi → validate files  # thiếu Origin = fail merge
Vì sao CI là sàn: rules advisory bị lờ ở mọi vendor non-Claude (verify 25/06); session hook lệch nhau + có lỗ. CI là nơi duy nhất nói "không" mà agent không sửa được khi merge.
05 · Cài đặt

06 · Mở rộng & giới hạn

Thêm luật/vendor · KNOWN GAPS · lên production

Mở rộng rẻ vì lõi tách khỏi adapter. Giới hạn được ghi rõ để biết vì sao cần CI.

Mở rộng

  • Thêm luật (cùng kind): sửa policy.yaml → gen-converters.py. Mọi vendor + CI cập nhật, không đụng code.
  • Thêm kind: thêm 1 handler check_<kind>() trong lõi.
  • Thêm vendor: ~15 dòng trong gen-converters.py.

Lên production

  • Gộp policy.yaml vào harness/policy.yaml
  • Đóng gói lõi thành release pin version → xoá copy editable + sync hash-3-mốc + bug remote_synced
  • Gọi gen từ install-harness.sh --vendor …
KNOWN GAPS (có chủ đích): lõi session soi BỀ MẶT lệnh bash → KHÔNG bắt python -c open(w), rm raw/, sed -i <script> raw/, ghi qua symlink. Đây KHÔNG phải bug — là lý do đảm bảo thật nằm ở CI (và sandbox nếu cần threat đối kháng). Threat hiện tại = agent hợp tác → CI đủ. test-broad.sh mục C assert đúng các gap này để đổi hành vi là biết ngay.
