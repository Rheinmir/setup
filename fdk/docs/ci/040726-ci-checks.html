00 · Tổng quan

Vì sao CI là "nơi nói KHÔNG" thật sự

Hook local (L1 Claude hook, L2 pre-commit) là advisory — agent non-Claude lờ được, người dùng --no-verify được. CI chạy trên máy sạch của GitHub, là cổng cuối trước merge. Cả 3 job đều dựng từ cùng lõi validator, không phụ thuộc vendor nào.

on Khi nào chạy

Mọi pull_request và mọi push lên nhánh orca. 3 job chạy song song, độc lập.

env Môi trường

ubuntu-latest · Python 3.x · chỉ pip install pyyaml (+ruff nếu có). Không cần Claude Code, không đọc .claude/settings.json.

L3 Tầng phòng thủ

L1 hook Claude · L2 pre-commit (bypass được) · L3 CI (đây) — không bypass khi merge.

02 · Job validate-content

validate file .md đổi (layer=repo)

Chỉ chạm các file .md thay đổi trong diff (so base PR / commit trước). Không có .md đổi → bỏ qua. Đây là nơi luật nội dung wiki (Origin, subfolder, frontmatter OKF…) được cưỡng chế trên chính thay đổi.

04 · Tóm tắt

Rút cục CI check gì — một bảng

Ba job, mục tiêu khác nhau: lõi còn chạy đúng · thay đổi .md hợp luật · toàn repo khỏe trên máy sạch.

JobKiểm gìSố cổng
selftestLõi vendor-neutral còn cắn đúng (demo + broad)67 assertion
validate-contentFile .md đổi hợp luật wiki (R1–R16)trên diff
repo-healthToàn repo khỏe: index/wikilink/retrieval-eval/rules/docs/leak16 cổng

Điều đáng nhớ: CI KHÔNG đọc .claude/settings.json (đó là config hook Claude, chạy local). Cổng "wiring policy→settings" đọc bản nguồn llmwiki/.claude/settings.json và fail-open — nên gitignore root settings.json không làm CI đỏ. Chỉ harness.yml buộc phải tracked vì GitHub Actions đọc workflow từ cây đã commit.
